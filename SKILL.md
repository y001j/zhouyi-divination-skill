# 周易三式占卜 · 起盘工具

这是一个把 Go 占卜程序封装成的工具型 skill。它**起盘排卦**并产出一份结构化的**解卦提示词**；拿到提示词后，**由你（调用方 agent）运用其中的卦象材料与解卦规则，亲自完成解读**。本 skill 不接模型、不替你解卦。

## 何时使用

当用户表达占卜意图时激活，典型信号：
- 「帮我算一卦」「起个卦」「占一下」「排个盘」「卜一卦」
- 「这件事吉凶如何」「该不该做某事」「问问运势/姻缘/财运」
- 明确点名某术：「用周易」「奇门看方位」「六壬断这个人」「三式互参」

如果用户没指定用哪种术，按下面「选术指引」替他挑，并说明理由。

## 四种术怎么选（选术指引）

各术所主不同，按问题性质选：

| 术（method） | 所主 / 所长 | 适合的问题 |
|---|---|---|
| `zhouyi` 周易六爻 | 明义理、辨吉凶之所以然、示进退取舍 | 该不该做、人生方向、心态决策、整体象意 |
| `qimen` 奇门遁甲 | 主天时地利——谋大局、定方位、择时机 | 选址方位、出行择时、布局谋略、竞争博弈 |
| `liuren` 大六壬 | 主人事情态——看人心、断曲折、定应期 | 具体人事、对方心意、寻人问事、何时应验 |
| `huican` 三式互参 | 周易×六壬×奇门 合参 | 重大或复杂之事，想多角度交叉印证 |

不确定时：人生抉择/心态类→`zhouyi`；方位时机类→`qimen`；人事情态/应期类→`liuren`；拿不准或大事→`huican`。

## 怎么调用

**第一步：拿到二进制路径**（脚本会按你的操作系统/架构准备好对应的预编译二进制，无需安装 Go）：

```bash
BIN=$(bash "${CLAUDE_SKILL_DIR}/scripts/ensure_binary.sh")
```

> `${CLAUDE_SKILL_DIR}` 是本 skill 目录。若该变量不可用，直接用脚本的绝对路径。
> 脚本会把诊断信息打到 stderr，stdout 只输出二进制路径，可安全用命令替换捕获。
> 脚本的查找顺序：① 本地 `bin/` 已有该平台二进制则直接用；② 否则从 GitHub Release 自动下载到 `bin/` 并缓存（首次需联网，之后离线）；③ 都不行则给出明确提示。支持 darwin-arm64/amd64、linux-amd64/arm64、windows-amd64 五个平台。

**第二步：起盘**（输出 JSON 到 stdout）：

```bash
"$BIN" cast --method <zhouyi|qimen|liuren|huican> \
            --question "用户所问之事" \
            --type <career|wealth|relation|health|decision|timing|other> \
            [--time 2026-05-30T14:30:00+08:00] \
            [--lon 116.4] \
            [--mode prompt|full] \
            [--upper N --lower N --changing N]
```

### 参数说明

- `--method, -m`：术数。`zhouyi` / `qimen` / `liuren` / `huican`（也接受中文 周易/奇门/六壬/互参）。默认 `zhouyi`。
- `--question, -q`：用户所问之事，原样传入。
- `--type, -t`：问题类型，决定提示词里的「侧重」段。可用键见下表，也接受中文（如「事业」「感情」）或数字 1–6。不传则归为 `other`。
- `--time`：起盘时刻，RFC3339 带时区（如 `2026-05-30T14:30:00+08:00`）。**不传则用当前时刻**——占卜讲究「即时起卦」，一般不传即可。
- `--lon`：经度（东经为正），按真太阳时校正。一般不需要。
- `--mode`：`prompt`（默认，仅起盘+提示词）或 `full`（额外附程序渲染的盘面/解卦文字到 `interpret` 字段，供你参考）。
- 周易专属（让卦象可复现，用于复盘/测试）：`--upper`（上卦数）`--lower`（下卦数）`--changing`（变爻位 0–6）。三者齐全且未加 `--coin` 时走数字起卦法，结果确定可复现；否则用铜钱法（随机）。`--coin` 强制铜钱法。

### 问题类型键

| 键 | 含义 |
|---|---|
| `career` | 事业 / 工作 |
| `wealth` | 财运 / 投资 |
| `relation` | 感情 / 姻缘 / 人际 |
| `health` | 健康 / 身心 |
| `decision` | 抉择 / 两难 |
| `timing` | 时运 / 整体吉凶 |
| `other` | 其他 / 不指定 |

## 输出格式

stdout 是一个 JSON 对象：

```json
{
  "ok": true,
  "method": "qimen",
  "question": "项目选址该往哪个方向",
  "questionType": "career",
  "questionLabel": "事业 / 工作",
  "time": "2026-05-30T14:30:00+08:00",
  "summary": "阳遁8局 · 值符天任落坤二宫 · 值使生门落坤二宫",
  "prompt": "你是一位精通奇门遁甲……（完整解卦提示词）"
}
```

- `ok`：是否成功。失败时为 `false`，`error` 字段含原因，进程退出码非 0（参数错误=2，起盘失败=1）。
- `summary`：一句话盘面摘要（卦名 / 课体 / 局数），可先报给用户。
- **`prompt`：核心交付物。** 里面已包含卦象/盘面全部材料、解卦规则、问题侧重、以及「请按以下结构解卦」的步骤。
- `interpret`：仅 `full` 模式有，程序渲染的盘面文字。

## 拿到结果后该做什么

1. （可选）先把 `summary` 报给用户，让他知道起了什么盘。
2. **把 `prompt` 当作你的解卦任务书**：按其中「请按以下结构解卦/解局/断课」的步骤，逐条解读，每个结论标注依据（提示词里要求了「（依据：…）」的格式）。
3. 注意各术的边界——提示词开头写明了「此术长于什么、不擅长什么」：
   - 周易：精确应期/方位非所长，无卦象支撑则不强断；
   - 奇门：幽微人心细务略带一笔；
   - 六壬：宏观方位国运点到为止。
   解读时遵守这些边界，不越术强断。
4. 用通俗语言把解读讲给用户，传统术语点到为止并解释。

## 注意事项

- **本 skill 只起盘、不解卦**。解读是你的工作，不要把 `prompt` 原样丢给用户——那是给你看的任务书。
- 占卜结果仅供参考与启发，不作为医疗、法律、投资等重大决策的唯一依据。涉及健康/法律/财务时，提醒用户咨询专业人士。
- 二进制按平台自动准备（本地 `bin/` 已有则用，否则从 GitHub Release 下载并缓存到 `bin/`），无需安装 Go。若 `ensure_binary.sh` 报「无法获得可用二进制」，多半是网络不通或你的平台不在支持的五个平台内——按脚本提示手动下载，或拿源码自行 `go build`。
- 同一时刻 + 同一参数，奇门/六壬/互参结果确定；周易默认铜钱法是随机的，要复现请用 `--upper/--lower/--changing`。

## 使用示例

```bash
# 1. 用户问「该不该接受外地新工作」——抉择类，用周易
BIN=$(bash "${CLAUDE_SKILL_DIR}/scripts/ensure_binary.sh")
"$BIN" cast -m zhouyi -q "该不该接受外地这份新工作" -t decision

# 2. 「新店开在哪个方向好」——方位类，用奇门
"$BIN" cast -m qimen -q "新店选址该往哪个方向" -t career

# 3. 「他到底有没有意思，什么时候会有结果」——人心+应期，用六壬
"$BIN" cast -m liuren -q "对方对我是否有意，何时有结果" -t relation

# 4. 「明年整体运势，重要的事想多方印证」——大事，用三式互参
"$BIN" cast -m huican -q "明年整体运势如何" -t timing
```
